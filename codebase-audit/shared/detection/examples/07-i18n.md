# Examples 07 — Internationalization (load on demand only)

FAIL / PASS / false-positive guards. Read a section only when a candidate is ambiguous.
Never flag logs/errors/debug/internal strings — only user-facing text.

---

## i18n_timezone

**FAIL — naive datetime stored/compared:**
```python
created = datetime.now()               # naive local time -> DST/region bugs
if token.expires < datetime.utcnow():  # naive comparison
```
**PASS — tz-aware UTC:**
```python
created = datetime.now(timezone.utc)
if token.expires < datetime.now(timezone.utc):
```
**FALSE-POSITIVE — monotonic timing / duration, not a stored timestamp:**
```python
start = time.monotonic(); ...; elapsed = time.monotonic() - start   # fine
```

## i18n_encoding

**FAIL — implicit encoding:**
```python
open("names.csv").read()               # platform-default codec; corrupts UTF-8 names
data.decode("ascii")                    # rejects accented chars
```
**PASS:**
```python
open("names.csv", encoding="utf-8").read()
data.decode("utf-8")
```

---

## i18n_hardcoded_strings

**FAIL — hardcoded UI text (i18n intent present):**
```jsx
<button>Add to cart</button>           // not translatable
```
**PASS:**
```jsx
<button>{t("cart.add")}</button>
```
**FALSE-POSITIVE — non-user-facing strings:**
```python
logger.info("cache miss for key %s", key)   # log, not UI
raise ValueError("invalid state")            # internal exception
STATUS = "PENDING"                            # machine enum key
```
Do not flag these. Also skip entirely if the app is single-locale-by-design (no i18n intent).

## i18n_locale_formatting

**FAIL — hardcoded format / currency:**
```js
`$${price.toFixed(2)}`                  // wrong symbol, decimal sep, grouping for most locales
d.strftime("%m/%d/%Y")                  // US-only date order
```
**PASS — locale-aware:**
```js
new Intl.NumberFormat(locale, { style: "currency", currency }).format(price)
new Intl.DateTimeFormat(locale).format(d)
```

---

## i18n_string_concat

**FAIL — sentence assembled from parts:**
```js
t("Hello") + " " + name + ", " + t("you have") + " " + n + " " + t("messages")
```
**PASS — one parameterized message:**
```js
t("greeting", { name, count: n })      // "Hello {name}, you have {count} messages"
```
